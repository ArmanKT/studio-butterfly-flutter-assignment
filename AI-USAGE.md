# AI Usage Transparency Log

This document records how AI tools were used during development, where they fell short, and what was done by hand.

---

## 1. My Workflow

I made all architectural decisions myself first, then built the project.

**Step 1 — I designed the architecture:**
* Chose BLoC for state management and wrote the ADR to justify it.
* Defined the clean feature-first folder structure (`domain`, `data`, `presentation`).
* Planned tenant isolation, token refresh flow, and pagination strategy before writing any code.

**Step 2 — I wrote the core logic by hand:**
* Wrote all four BLoCs (`TenantBloc`, `SmsSendBloc`, `MessageHistoryBloc`, `CostBreakdownBloc`) myself.
* Built the custom `Money` class using integer micro-cents math to avoid floating-point billing errors.
* Wrote the token auto-refresh interceptor and the background delivery status polling engine.

**Step 3 — I used AI for speed and optimization:**
* Used AI to write UI widget layouts (forms, cards, lists) based on my design directions.
* Used AI to suggest and apply Material 3 theme optimizations.
* Described the full mock requirements to AI and it wrote `MockHttpClient` from those instructions.
* Used AI to scaffold initial test case structures, which I then reviewed, fixed, and extended.

**Step 4 — I tested and fixed everything manually:**
* Ran the app on both Android emulator and macOS desktop.
* Fixed bugs including layout overflows, pagination state issues, and billing display errors.
* Verified tenant switching, error states (429, 502), and empty state flows by hand.

## 2. Where AI was wrong

### Theme Properties
* **Problem:** AI used deprecated `ColorScheme.background` and wrong `cardTheme` setup.
* **Fix:** Replaced with correct Material 3 parameters (`surface`, `CardThemeData`).

### HTTP Header Case Mismatch
* **Problem:** AI wrote `Retry-After` in the mock but looked for `retry-after` in the parser. Dart maps are case-sensitive, so the header was never found.
* **Fix:** Made header lookup case-insensitive.

### Layout Overflow on Small Screens
* **Problem:** AI put a long text title and icon button in an unconstrained `Row`. On a 360px screen it overflowed.
* **Fix:** Wrapped the title in `Expanded` so it wraps correctly.

### Billing Math Precision
* **Problem:** AI used `double` arithmetic for cost totals, which produces errors like `$0.30000000000000004`.
* **Fix:** Rewrote billing math using integer micro-cents inside the `Money` class.

### Pagination Load-More Logic
* **Problem:** AI generated a naive pagination loader that fetched the next page incorrectly. It did not preserve the cursor state properly, causing duplicate results or missed pages on scroll.
* **Fix:** Rewrote the pagination state handling in `MessageHistoryBloc` to correctly track and pass the cursor across page loads.

### BLoC Initial Data Not Loading on First Launch
* **Problem:** AI did not add an initial fetch event in the `CostBreakdownBloc` constructor. The billing card showed empty on first launch because no data was ever requested.
* **Fix:** Added the initial `FetchCostBreakdown` event dispatch inside the BLoC constructor so data loads automatically on startup.

### General Logic Errors in AI-Generated BLoC Code
* **Problem:** Several AI-generated state transition handlers had incorrect conditions — for example, emitting a success state before awaiting an async call, or not clearing stale data when the tenant switched.
* **Fix:** Manually reviewed every BLoC handler, corrected the async flow, and added tenant-switch cleanup logic to prevent data leaking between tenants.

## 3. What I wrote entirely by hand

I wrote the core business logic myself from scratch. AI was used for structure and speed, but the critical parts were my own work.

* All BLoC events, states, and handler logic.
* The `Money` minor-units class.
* All domain models (`SmsMessage`, `CostBreakdown`, `Tenant`, `Money`), repository interfaces, and custom API exception classes.
* The token refresh and retry interceptor.
* The background delivery status polling timer.
* The initial test case structures were generated by AI, but I reviewed every test, fixed the failing ones, and corrected logic errors in the mocks and assertions.
